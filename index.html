How it works

Declare, stress, reconcile.

01

Architecture model

Declare what the system says it is. Providers, regions, agents, dependencies, storage tiers, recovery targets, and criticality class. This becomes the source of truth inside MIRRORNODE.

02

Stress engine

Run fault scenarios against the declared model. The model is deterministic — same inputs, same outputs. Fault scenarios layer probabilistic estimates on top of that model; treat those as directional.

03

Drift detector

Compare the declared model against other declarations: infrastructure-as-code, tracked work, runbooks, and deployment records. Drift detection compares declarations, not live infrastructure — it won't catch config that exists only in production.

Evidence review demo

Drift review surface

The drift review surface compares a declared architecture manifest against other declarations when configured. It is designed to support manual review, not replace formal audit or operational verification.
